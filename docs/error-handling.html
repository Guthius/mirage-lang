Error Handling

The error type, try propagation, and defer cleanup.

The error Type

The error type is a 64-bit unsigned integer. By convention:

  • 0 — no error (success)
  • any non-zero value — an error code

There is no built-in error enum or string. Error codes are application-defined.

Fallible Functions

A function is fallible when its last return type is error. Such functions return both a result value and an error code simultaneously. 

# Returns (quotient, error): 0 = success, 1 = division by zero
fn divide(a: i32, b: i32) -> (i32, error) {
    if b == 0 {
        return 0, 1   # error code 1
    }
    return a / b, 0
}

# Void-return fallible
fn open_file(path: *u8) -> error {
    # ... attempt to open ...
    return 0
}

Must Capture Errors

Ignoring the return value of a fallible function is a compile-time error. You must either capture the error or use try. 

# Correct: capture all return values
mut result, err := divide(10, 2)
if err != 0 {
    printf("error: %llu\n", err)
    return err
}
printf("result = %d\n", result)

The try Expression

try is a unary operator that handles the error return of a fallible call. If the error is non-zero, try immediately returns the error from the enclosing function. The enclosing function must itself return error as its last type. 

fn process() -> error {
    # If divide fails, propagate the error up immediately
    mut q: i32 = try divide(10, 2)   # q = 5

    # Chained: try in a group decl
    mut q2, e2 := divide(20, 4)
    if e2 != 0 { return e2 }

    printf("%d %d\n", q, q2)
    return 0
}

pub fn main() -> error {
    # Propagate any error from process() up to OS
    try process()
    return 0
}

Try Precedence

try binds tighter than binary operators: 

# try applies only to f(x), not to the entire expression
try f(x) + g()    # same as: (try f(x)) + g()

# Parentheses to chain member access after try:
const s := (try lookup(key)).size

The defer Statement

defer registers a statement to run when the enclosing block exits — whether by fall-through, return, break, or continue. 

fn with_resource() -> error {
    const handle := open("file.txt")
    defer close(handle)   # always runs on exit

    try read(handle)      # defer still runs if this propagates
    return 0
}

LIFO Order

Multiple defers in the same scope run in last-in, first-out order (like a stack unwind):

fn lifo_demo() {
    defer printf("first registered\n")
    defer printf("second registered\n")
    defer printf("third registered\n")
}
# Output:
# third registered
# second registered
# first registered

Defer in Nested Blocks

Each block has its own defer scope. Inner-block defers fire when that block exits:

fn block_defer() {
    {
        defer printf("inner\n")    # fires when inner block exits
    }
    printf("outer\n")              # runs after inner defer
}
# Output:
# inner
# outer

Defer in Loops

A defer inside a loop body fires on every iteration, including on break:

mut i: i32 = 0
while i < 3 {
    defer printf("tick\n")   # prints "tick" once per iteration
    i++
}
# Output: tick tick tick

Defer and Early Return

Defers run even on early return. Return values are captured before defers execute:

fn example() -> i32 {
    defer printf("cleanup\n")
    if some_condition {
        return 1   # defer fires before the ret instruction
    }
    return 0
}

Defer Restrictions

  • Using try inside a defer body is a compile error.
  • return, break, or continue that escape the defer body are compile errors.

Defer + Try Together

When try propagates an error, all deferred statements accumulated up to that point in the function run before the function returns. This enables reliable cleanup even in error paths: 

fn acquire_and_process() -> error {
    const a := try acquire_resource_a()
    defer release_resource_a(a)

    const b := try acquire_resource_b()
    defer release_resource_b(b)

    # If this fails, both release_b and release_a will run (LIFO)
    try do_work(a, b)

    return 0
}

Error in Entry Points

The error type can be the return type of main. The value is truncated to i32 and used as the process exit code:

pub fn main() -> error {
    try some_init()
    try run()
    return 0
}

Full Example

ext fn printf(fmt: *u8, ...) -> i32

fn maybe_fail(should_fail: bool) -> error {
    if should_fail { return 42 }
    return 0
}

fn divide(a: i32, b: i32) -> (i32, error) {
    if b == 0 { return 0, 1 }
    return a / b, 0
}

fn test_defer_before_try() -> error {
    defer {
        printf("cleanup ran\n")
    }
    try maybe_fail(true)    # propagates 42; defer runs first
    return 0
}

pub fn main() -> i32 {
    const e: error = test_defer_before_try()
    printf("error code: %llu\n", e)   # 42

    mut q, err := divide(10, 0)
    if err != 0 {
        printf("division failed\n")
        return 1
    }
    return 0
}
