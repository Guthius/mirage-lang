Control Flow

Conditionals, loops, and pattern matching.

If / Else

The if statement evaluates a bool condition and executes the then branch if true.

if x > 0 {
    printf("positive\n")
} else if x < 0 {
    printf("negative\n")
} else {
    printf("zero\n")
}
  • The condition must be exactly bool — no implicit integer-to-bool coercion.
  • The else branch is optional.
  • The body can be any statement, including a non-block single statement.

Ternary Expression

For simple conditional values, use the ternary operator:

const abs: i32 = x < 0 ? 0 - x : x

While Loop

while is the only looping construct in Mirage. It repeatedly executes the body block as long as the condition is true. 

mut i: i32 = 0
while i < 10 {
    printf("%d\n", i)
    i++
}

Infinite Loop

while true {
    # runs forever until break
    if done { break }
}

Loop with Decrement Condition

Since -- is a post-decrement expression, it can be used directly as the while condition:

mut n: usize = 5
while n-- {
    # runs 5 times: n is tested before decrement
}

Break and Continue

mut i: i32 = 0
while i < 10 {
    i++
    if i == 3 { continue }  # skip 3
    if i == 7 { break }     # stop at 7
    printf("%d\n", i)
}
Defer in Loops
A defer inside a loop body fires on every iteration (including on break and continue). See Error Handling.

Match Expression

match is an expression — it has a result value. All arms must produce the same type. Match is exhaustive: the compiler checks that all possible values are covered.

Matching on Enums

type Color = enum(i32) {
    red
    green
    blue
}

fn color_name(c: Color) -> *u8 {
    return match c {
        .red:   "red",
        .green: "green",
        .blue:  "blue"
    }
}

Matching on Integers

Integer match requires a _ default arm (exhaustiveness cannot be verified for large domains):

fn classify(x: i32) -> *u8 {
    return match x {
        0:  "zero",
        1:  "one",
        -1: "minus one",
        _:  "other"
    }
}

Matching on Booleans

Boolean match is exhaustive when both true and false are covered:

fn bool_label(b: bool) -> i32 {
    return match b {
        true:  1,
        false: 0
    }
}

Matching on Tagged Unions

type Shape = union(enum) {
    circle: struct { radius: f64 }
    rect:   struct { w: f64; h: f64 }
    point
}

fn area(s: Shape) -> f64 {
    return match s {
        .circle(c): c.radius * c.radius * 3.14159,
        .rect(r):   r.w * r.h,
        .point:     0.0
    }
}

# By-reference capture: s is *PayloadStruct
const desc := match shape {
    .circle(&c): c.radius,
    .rect(r):   r.w,
    .point:     0.0
}

Default / Wildcard Arm

Use _ as the last arm to match any remaining value. For enums, a _ after full coverage is a compile error. _ must always be last. 

match code {
    200: "ok",
    404: "not found",
    _:   "other"
}

Switch Statement

switch is the statement-level counterpart to match. Arm bodies are statements (not expressions). There is no exhaustiveness requirement and no result type. 

mut result: i32 = 0
switch x {
    0: result = 10,
    1: result = 20,
    _: result = 99
}

Switch arms use the same pattern syntax as match: variant patterns (.name), literal patterns, and the default wildcard _.

Switch on Enum

switch color {
    .red:   printf("red\n"),
    .green: printf("green\n"),
    .blue:  printf("blue\n")
}

Switch on Tagged Union

switch shape {
    .circle(c): printf("circle r=%.1f\n", c.radius),
    .rect(r):   printf("rect %.1fx%.1f\n", r.w, r.h),
    .point:     printf("point\n")
}
Switch vs Match
Use match when you need a result value (it's an expression). Use switch when you need to execute statements per arm. Only match checks exhaustiveness.
