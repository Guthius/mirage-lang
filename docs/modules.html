Modules

Splitting code across files with import and pub.

Importing a Module

A module is a .mir source file. You import it using the import expression, which must appear as the initializer of a const declaration with no explicit type: 

const mem := import("mem")
const vec := import("vector")

The string is a path (without the .mir extension) resolved relative to the compiler's search paths. The import result is a namespace binding — it is not a value and cannot be assigned, passed as an argument, or stored.

Accessing Module Symbols

After importing, you access the module's public symbols through the namespace name using dot notation: 

const mem := import("mem")

fn example() {
    mut ptr: anyptr = mem.malloc(64)     // call a function
    mem.free(ptr)
    mut v: mem.SomeType                    // use a type
}

Visibility with pub

Declarations in a module are private by default. To make them accessible from other modules, add pub: 

// vector.mir

const mem := import("mem")

// Public — visible outside this module
pub type vector = struct {
    data:         anyptr
    length:       usize
    capacity:     usize
    element_size: usize
}

pub fn init(v: *vector, element_size: usize) -> bool {
    v.data         = nil
    v.length       = 0
    v.capacity     = 0
    v.element_size = element_size
    return true
}

// Private — not accessible from importing modules
fn reserve(v: *vector, cap: usize) -> bool {
    // ...
    return true
}

What Can Be pub?

DeclarationExample
Functionpub fn name()

Methods and Modules

Methods on a type are visible cross-module only if they are individually marked pub. The impl block itself cannot be pub. 

// vector.mir
impl vector {
    pub fn push(self, element: anyptr) -> bool {
        // ...
    }

    pub fn at(self, idx: usize) -> anyptr {
        // ...
    }
}

// start.mir
const vec := import("vector")

pub fn main() -> i32 {
    mut v: vec.vector
    vec.init(&v, sizeof(v))

    const x: i32 = 42
    v.push(&x)   // method call on cross-module type

    const result := *cast(v.at(0), *i32)
    return result
}

Chained Module Access

If a module re-exports another module via pub const, you can chain access:

// lib.mir
pub const utils := import("utils")

// main.mir
const lib := import("lib")

pub fn main() {
    lib.utils.do_something()
}

Global Variables

Top-level mut and const declarations are module-level globals. They live for the lifetime of the program and are accessible from anywhere in the same module (or cross-module if pub). 

// mem.mir
mut heap_head: *block_header = nil
mut heap_tail: *block_header = nil

pub mut allocation_count: u64 = 0

pub const alignment: usize = 16

Complete Module Example

Here is a minimal two-file example. math.mir provides arithmetic utilities; main.mir imports and uses them.

// math.mir

pub fn abs(x: i32) -> i32 {
    return x < 0 ? 0 - x : x
}

pub fn clamp(x: i32, lo: i32, hi: i32) -> i32 {
    if x < lo { return lo }
    if x > hi { return hi }
    return x
}
// main.mir

ext fn printf(fmt: *u8, ...) -> i32
const math := import("math")

pub fn main() {
    printf("%d\n", math.abs(-5))           // 5
    printf("%d\n", math.clamp(15, 0, 10))  // 10
}
