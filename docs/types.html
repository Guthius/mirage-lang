Types

Mirage's type system: primitives, pointers, arrays, slices, and function pointers.

Integer literals can be written in decimal, hex (0x), or binary (0b). Underscores are allowed as separators:

const a := 1_000_000
const b := 0xFF
const c := 0b1010_1010
No Implicit Coercion
Mirage does not promote or widen integer types automatically. Mixing types in arithmetic is a compile error. Use cast(expr, TargetType) to convert.

Boolean Type

The bool type has two values: true and false. It is 1 byte.

const flag: bool = true
const result := x > 0   # comparison produces bool

The byte Type

byte is an alias for u8. It is conventionally used for raw byte buffers:

mut buf: [512]byte
mut p: *byte = cast(ptr, *byte)

The error Type

error is an unsigned 64-bit integer. By convention, 0 means success and any non-zero value is an error code. See Error Handling for details.

The anyptr Type

anyptr is an untyped pointer — equivalent to void* in C. It can hold any pointer value and supports arithmetic with integer operands. All typed pointers are implicitly assignable to and from anyptr. 

mut raw: anyptr = nil
raw = cast(my_struct_ptr, anyptr)
mut next: anyptr = raw + 16    # pointer arithmetic

Pointer Types

A pointer type *T points to a value of type T.

mut x: i32 = 42
const p: *i32 = &x    # address-of
const v: i32 = *p   # dereference
*p = 99              # write through pointer

Auto-deref: When accessing a struct field or calling a method on a pointer-to-struct, the pointer is automatically dereferenced. There is no need for ->: 

const ptr: *Vec2 = &v
printf("%f\n", ptr.x)       # same as (*ptr).x
ptr.length_sq()              # method call through pointer

Array Types

An array [N]T holds exactly N elements of type T. The size must be a compile-time constant.

mut arr: [5]i32 = {1, 2, 3}   # partial init; remaining elements zeroed
const n: usize = len(arr)      # 5
const sz: usize = sizeof(arr)   # 20 bytes
arr[0] = 99                      # index access

Arrays are value types — assigning an array copies all elements.

Array Initialization

# No init: all elements default-initialized to 0
mut a: [3]i32

# Partial: first 2 set, remaining zeroed
mut b: [3]i32 = {1, 2}

# Full
mut c: [3]i32 = {10, 20, 30}

# Empty braces: same as no init
mut d: [3]i32 = {}

Slice Types

A slice []T is a (pointer, length) pair. It does not own memory. Slices are 16 bytes (pointer + 64-bit length). 

mut arr: [5]i32 = {1, 2, 3, 4, 5}

# Slice from array (implicit coercion)
mut s: []i32 = arr

# Slice sub-range
const sub := arr[1..3]     # elements [1] and [2]

# Slice from pointer + length
const ps := cast(ptr, []u8, 64)

const n: usize = len(s)    # length

Function Pointer Types

Function pointer types describe callable values. They are opaque 8-byte pointers internally. 

# Type syntax
fn(i32, i32) -> i32
fn(i32) -> (i32, error)      # multi-return
fn(*u8, ...) -> i32           # variadic
fn()                           # void, no params
fn add(a: i32, b: i32) -> i32 { return a + b }
fn mul(a: i32, b: i32) -> i32 { return a * b }

# Store a function pointer
mut fp: fn(i32, i32) -> i32 = add
fp(3, 4)   # returns 7

fp = mul
fp(3, 4)   # returns 12

# nil / default are valid null function pointers
mut nullable: fn(i32, i32) -> i32 = nil
if nullable == nil {
    printf("null\n")
}

Function Pointers in Structs

type Transform = struct {
    f: fn(i32) -> i32
    label: *u8
}

mut t: Transform
t.f = square
const result: i32 = t.f(5)   # call via struct field

Casting Function Pointers

Function pointers can be cast to/from anyptr for C interop:

mut raw: anyptr = cast(fp, anyptr)
mut fp2: fn(i32) -> i32 = cast(raw, fn(i32) -> i32)

The cast Operator

All type conversions are explicit in Mirage. The cast operator converts a value to a target type. 

cast(expr, TargetType)
cast(ptr, []T, length)    # pointer → slice

Valid cast combinations:

  • Any scalar type ↔ any scalar type (integers, floats, bool, anyptr, pointers)
  • anyptr ↔ any typed pointer
  • anyptr ↔ any function pointer type
  • Array [N]T → slice []T
  • Pointer or anyptr → slice with length: cast(p, []T, n)

The sizeof and len Operators

const sz: usize = sizeof(my_struct)   # size of a value or named type
const n:  usize = len(my_array)        # element count of array or slice
Note
sizeof cannot take built-in type keywords directly (e.g. sizeof(i32) is unsupported). Use a variable of that type: sizeof(my_i32_variable).
