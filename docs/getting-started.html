Getting Started

Build Mirage from source and write your first program.

Building the Compiler

Mirage requires LLVM and a C++ compiler. The project uses just as its build tool.

# Clone and build
git clone <repo-url>
cd Mirage
just build

The compiler binary is produced at ./build/mirage.

Compiler Flags

FlagDescription
--emit-irPrint the generated LLVM IR to stdout instead of producing a binary
--freestandingBuild without a hosted entry point; use fn _start() instead of main
-o <file>Set the output file name
# Compile a program
./build/mirage hello.mir -o hello

# Inspect the generated LLVM IR
./build/mirage hello.mir --emit-ir

# Freestanding (no libc, custom entry)
./build/mirage kernel.mir --freestanding -o kernel.o

Hello, World

Mirage does not have a built-in print function. Instead, it calls the C standard library's printf via an ext fn declaration. This is typical for Mirage programs that run in a hosted environment. 

# hello.mir

ext fn printf(fmt: *u8, ...) -> i32

pub fn main() {
    printf("Hello, World!\n")
}
./build/mirage hello.mir -o hello
./hello
# Hello, World!

Entry Points

The root module must define a pub fn main in one of three signatures: 

# No return value — exits with code 0
pub fn main() {
    # ...
}

# Returns an explicit exit code
pub fn main() -> i32 {
    return 0
}

# Returns an error value (truncated to i32 for the exit code)
pub fn main() -> error {
    return 0
}

Basic Program Walkthrough

Let's walk through a small but complete Mirage program that shows variables, conditionals, loops, and C interop. 

ext fn printf(fmt: *u8, ...) -> i32

fn fizzbuzz(n: i32) {
    mut i: i32 = 1
    while i <= n {
        if i % 15 == 0 {
            printf("FizzBuzz\n")
        } else if i % 3 == 0 {
            printf("Fizz\n")
        } else if i % 5 == 0 {
            printf("Buzz\n")
        } else {
            printf("%d\n", i)
        }
        i++
    }
}

pub fn main() {
    fizzbuzz(20)
}

Key points in this example:

  • ext fn printf — declares the C printf function as a variadic extern.
  • mut i: i32 = 1 — declares a mutable variable with an explicit type.
  • while i <= n { ... } — the only loop construct in Mirage.
  • i++ — post-increment expression.
  • i % 15 — modulo operator.

Variables and Constants

# Mutable variable — type inferred
mut x := 42

# Mutable variable — explicit type
mut y: f64 = 3.14

# Immutable constant — cannot be reassigned
const limit := 100

# Default initialization — struct/array fields zeroed
mut buf: [256]u8

# Uninitialized — storage allocated, no write
mut tmp: i32 = undefined
tmp = 99

Type Inference

Mirage infers the type of a variable from its initializer when no annotation is given. The := syntax means "infer the type": 

const a := 10         # a: i32
mut   b := 3.14       # b: f64
const c := true        # c: bool
const s := "hello"    # s: *u8
Note
Mirage does not implicitly promote or coerce numeric types. To mix types in arithmetic, use cast(expr, TargetType).
