Structs & Types

Structs, enums, tagged and untagged unions, impl blocks.

Type Declarations

Named types are declared with the type keyword:

type Point = struct {
    x: f64
    y: f64
}

pub type Color = enum(u8) {
    red
    green
    blue
}

Add pub to make the type visible to importing modules.

Structs

Structs are product types — they hold named fields of (potentially different) types. Fields are separated by newlines in the definition.

type Vec2 = struct {
    x: f64 = 0.0    // field-level default
    y: f64 = 0.0
}

Creating Struct Values

// No init: all fields use their defaults (or zero)
mut a: Vec2

// Partial init: x overridden, y uses its default (0.0)
mut b: Vec2 = {x = 5.0}

// Full init
mut c: Vec2 = {x = 3.0, y = 4.0}

// Empty braces: same as no init
mut d: Vec2 = {}

// 'default' keyword: explicitly default-initialize
mut e: Vec2 = default

Accessing Fields

const len := c.x * c.x + c.y * c.y

// Through pointer: auto-dereference
const p: *Vec2 = &c
printf("%f\n", p.x)   // same as (*p).x

Packed Structs

A packed struct has no alignment padding between fields:

type Header = struct(packed) {
    magic:   u32
    version: u8
    flags:   u16
}

Enums

Enums are integer-backed named constants. The underlying integer type is specified in parentheses; if omitted, it defaults to i32. 

type Direction = enum(u8) {
    north
    south
    east
    west
}

Custom Values and Iota

Fields are assigned values starting at 0 by default. Use explicit initializers to override. The iota keyword represents the field's sequential index (0, 1, 2, …): 

type Flag = enum(u32) {
    none  = 0
    read  = 1
    write = 2
    exec  = 4
}

// iota used as a power-of-two template
type Bits = enum(u32) {
    a = 1 << iota    // template: 1<<0=1, 1<<1=2, 1<<2=4
    b
    c
}

Using Enum Values

// Contextual (type inferred from annotation)
const d: Direction = .north

// Qualified
const d2 := Direction.south

// Match on enum
const label := match d {
    .north: "N",
    .south: "S",
    .east:  "E",
    .west:  "W"
}

Untagged Unions

Untagged unions overlay all members at offset 0. The size is the size of the largest member rounded up to the maximum alignment. All members share the same storage. 

type NumUnion = union {
    as_i64: i64
    as_f64: f64
    as_u8:  u8
}

// Initialize with exactly one member
mut u: NumUnion = {as_i64 = 42}

// Read/write any member
u.as_f64 = 3.14
const bits: i64 = u.as_i64   // reinterpret bits

// Undefined init is legal
mut v: NumUnion = undefined
v.as_i64 = 100
Restrictions
  • Exactly one member must be named in the braced initializer (0 or 2+ is a sema error).
  • Member default initializers are not allowed in union definitions.
  • default and {} are not valid initializers for untagged unions.
  • Untagged unions cannot be used in ext fn parameter or return types.

Tagged Unions

Tagged unions (union(enum)) are discriminated unions — a runtime tag identifies which variant is active. They are the safe alternative to untagged unions for sum types. 

type Shape = union(enum) {
    circle: struct { radius: f64 }
    rect:   struct {
        w: f64
        h: f64
    }
    point              // payload-free variant
}

Layout

A tagged union always contains a u32 discriminant tag at offset 0. The payload follows at an alignment-padded offset. The overall size equals align_up(payload_offset + max_payload_size, align).

Constructing Tagged Union Values

// Qualified form: TypeName.variant{.field = val}
const c: Shape = Shape.circle{.radius = 3.0}

// Contextual form: .variant{.field = val} (type from annotation)
mut r: Shape = .rect{.w = 4.0, .h = 5.0}

// Payload-free variant
const p: Shape = .point

Destructuring with Match

Tagged union members cannot be accessed directly — use match:

const a: f64 = match s {
    .circle(c): c.radius * c.radius * 3.14159,
    .rect(r):   r.w * r.h,
    .point:     0.0
}

// By-reference capture: c is *PayloadStruct (no copy)
match s {
    .circle(&c): c.radius = 10.0,    // mutate in-place
    .rect(&r):   r.w    = 0.0,
    .point:     0.0
}

Impl Blocks

impl blocks add methods to a named type (struct or enum). Methods receive self — an implicit pointer to the receiver — as their first parameter. 

type Counter = struct {
    value: i32
}

impl Counter {
    // Read-only method: self is implicitly *Counter (const view)
    pub fn get(self) -> i32 {
        return self.value
    }

    // Mutating method: mut self allows field writes
    pub fn increment(mut self) {
        self.value++
    }

    pub fn add(mut self, n: i32) -> i32 {
        self.value += n
        return self.value
    }
}

Calling Methods

mut c: Counter
c.value = 0
c.increment()
const v: i32 = c.get()        // 1
const v2: i32 = c.add(5)      // 6

// Calling through a pointer: auto-deref
const ptr: *Counter = &c
ptr.increment()

Impl Rules

  • impl blocks cannot be marked pub; use pub on individual methods instead.
  • Methods are defined in the same module as the type (cross-module impl is not supported).
  • self is internally a pointer; field access auto-derefs.
  • Methods cannot be stored in function pointers because their first argument is the implicit receiver pointer.

Methods on Enums

type Direction = enum(u8) { north  south  east  west }

impl Direction {
    pub fn opposite(self) -> Direction {
        return match *self {
            .north: .south,
            .south: .north,
            .east:  .west,
            .west:  .east
        }
    }
}

Default and Undefined Initializers

default

Produces the type's default zero value. For structs with field defaults, those defaults are applied recursively.

mut x: i32    = default   // 0
mut v: Vec2   = default   // {x=0.0, y=0.0} (or field defaults)
mut fp: fn(i32) -> i32 = default  // nil

undefined

Allocates storage but does not initialize it. Useful for performance-sensitive code where you immediately assign a value.

mut buf: [1024]byte = undefined
// Must initialize before reading buf!
fill_buffer(&buf[0], len(buf))
Warning
undefined is not valid in const declarations. Reading an uninitialized value before assignment is undefined behavior.
