Mirage

A compiled, statically-typed systems language targeting native code via LLVM IR. Designed for low-level programming with a clean, expression-oriented syntax.

Get Started Grammar Reference

What is Mirage?

Mirage is a systems programming language that compiles directly to native machine code through LLVM. It features a type system built around value semantics, explicit mutability, and C interoperability via ext fn declarations — without a garbage collector or runtime.

The language provides fine-grained control over memory layout while keeping the syntax readable. Error handling is built into the type system via the error type and the try/defer mechanism.

Key Features

Native Compilation

Compiles to machine code via LLVM. Produces small, fast executables with no runtime overhead.

Strong Static Typing

Every value has a known type at compile time. No implicit numeric promotions — casts are explicit.

Explicit Mutability

All variables are immutable by default (const). Mutable state requires mut.

C Interop

Call any C library function with ext fn declarations, including variadic functions.

Algebraic Types

Structs, enums, tagged unions, and untagged unions with exhaustive match.

Error Handling

First-class error type with try propagation and defer cleanup.

Function Pointers

First-class function pointer types with full type safety and C callback interop.

Module System

Explicit module imports with pub visibility control. No header files.

Quick Look

A taste of Mirage syntax — a simple program using structs, methods, and error handling:

ext fn printf(fmt: *u8, ...) -> i32

type Vec2 = struct {
    x: f64
    y: f64
}

impl Vec2 {
    pub fn length_sq(self) -> f64 {
        return self.x * self.x + self.y * self.y
    }
}

fn divide(a: f64, b: f64) -> (f64, error) {
    if b == 0.0 { return 0.0, 1 }
    return a / b, 0
}

pub fn main() -> error {
    mut v: Vec2 = {x = 3.0, y = 4.0}
    printf("length_sq = %.1f\n", v.length_sq())

    mut result: f64 = try divide(v.x, v.y)
    printf("x / y = %.4f\n", result)

    return 0
}
