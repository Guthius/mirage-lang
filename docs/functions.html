Functions

Declaring, calling, and passing functions in Mirage.

Basic Function Declarations

fn add(a: i32, b: i32) -> i32 {
    return a + b
}

# No return value (void)
fn greet(name: *u8) {
    printf("Hello, %s!\n", name)
}

Mutable Parameters

Function parameters are immutable by default. Declare a parameter mut to make it mutable inside the function:

fn countdown(mut n: i32) {
    while n > 0 {
        printf("%d\n", n--)
    }
}
Note
Making a parameter mut does not affect callers — Mirage passes by value. To modify the caller's variable, pass a pointer.

Multi-Return Functions

Functions can return multiple values. The return types are listed in parentheses after ->. Callers use group declarations to capture them. 

fn divmod(a: i32, b: i32) -> (i32, i32) {
    return a / b, a % b
}

mut q, r := divmod(17, 5)   # q=3, r=2

Discarding Return Values

Use _ to discard one or more return values:

mut q, _ := divmod(17, 5)   # only keep quotient

Fallible Functions (Error Return)

By convention, a function that can fail has error as its last return type. Callers must capture the error or use try. 

fn divide(a: i32, b: i32) -> (i32, error) {
    if b == 0 { return 0, 1 }
    return a / b, 0
}

# Must capture error
mut result, err := divide(10, 2)
if err != 0 { /* handle */ }

# Or propagate with try (enclosing fn must return error)
mut val: i32 = try divide(10, 2)

See Error Handling for full details.

Visibility: pub

By default, functions are module-private. Add pub to make them accessible from other modules: 

pub fn compute(x: i32) -> i32 {
    return x * 2
}

# Private — only callable within this module
fn helper(x: i32) -> i32 {
    return x + 1
}

Extern Functions

Use ext fn to declare external C functions. The declaration provides the Mirage-visible type signature; the linker resolves the actual symbol. 

# C standard library declarations
ext fn puts(s: *u8) -> i32
ext fn malloc(size: usize) -> anyptr
ext fn free(ptr: anyptr)
ext fn memcpy(dest: anyptr, src: anyptr, n: usize) -> anyptr

Variadic Extern Functions

Add ... at the end of the parameter list to declare a variadic extern function. At least one named parameter must come before .... 

ext fn printf(fmt: *u8, ...) -> i32
ext fn snprintf(buf: *u8, n: u64, fmt: *u8, ...) -> i32

printf("%d %f\n", 42, 3.14)
Variadic Argument Types
Variadic arguments must be at least 32 bits wide and match C's default argument promotion rules. Valid types: i32, u32, i64, u64, usize, error, f64, typed pointers, anyptr. Narrower types (e.g. f32, u8) must be explicitly cast first.

Entry Points

The root module must define exactly one of:

pub fn main() { /* ... */ }                    # exits with code 0
pub fn main() -> i32 { return 0 }                # returns exit code
pub fn main() -> error { return 0 }              # error truncated to i32

For freestanding builds (--freestanding), define fn _start() instead. The pub modifier is not required for _start.

Function Pointers

Functions are first-class values. They can be stored in variables, passed to functions, and stored in structs.

Storing and Calling

fn square(x: i32) -> i32 { return x * x }

mut fp: fn(i32) -> i32 = square
const r: i32 = fp(9)   # 81

Higher-Order Functions

fn apply(op: fn(i32, i32) -> i32, x: i32, y: i32) -> i32 {
    return op(x, y)
}

const sum := apply(add, 3, 4)   # 7

Variadic Function Pointer

ext fn printf(fmt: *u8, ...) -> i32

mut vfp: fn(*u8, ...) -> i32 = printf
vfp("%d\n", 42)

Multi-Return Function Pointer

fn divide(a: i32, b: i32) -> (i32, error) {
    if b == 0 { return 0, 1 }
    return a / b, 0
}

mut fp: fn(i32, i32) -> (i32, error) = divide
mut val, err := fp(10, 2)

# try also works through fn ptrs
mut v: i32 = try fp(8, 2)

Macros

Macros are expression-level compile-time substitutions. They look like function calls but expand to an inline expression. 

macro max(a: i32, b: i32) -> a > b ? a : b
macro align_up(n: usize) -> (n + (15)) & ~(usize)15

const m := max(3, 7)        # expands to: 3 > 7 ? 3 : 7 → 7
const aligned := align_up(size)

Macros can be exported with pub:

pub macro min(a: i32, b: i32) -> a < b ? a : b
Macros vs Functions
Macros are syntactic substitution — they cannot be stored as function pointers and cannot recurse. They are ideal for small, performance-critical inline expressions.
